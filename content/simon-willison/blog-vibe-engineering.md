# Vibe Engineering

**Date:** October 7, 2025

## Overview

Simon Willison proposes "vibe engineering" as a term for the productive, accountable use of LLMs by experienced software professionals—contrasting it with "vibe coding," which he describes as "fast, loose and irresponsible" AI-driven development.

## Key Distinction

Willison argues that working effectively with coding agents requires maintaining responsibility for the produced code, unlike vibe coding where results are accepted passively. He notes that "it's _difficult_" to use LLMs productively on real projects, requiring substantial expertise.

## Engineering Practices That Excel With AI

Willison identifies twelve practices that become increasingly valuable:

- Robust automated testing (agents perform optimally with comprehensive test suites)
- Advance planning and high-level design
- Comprehensive documentation
- Strong version control habits
- Effective automation infrastructure
- Active code review culture
- Meticulous manual QA beyond automated tests
- Research skills for evaluating solutions
- Safe preview environments for features
- Judgment about what tasks suit AI
- Updated project estimation abilities
- A "weird form of management" comparable to directing human collaborators

## Core Thesis

"AI tools **amplify existing expertise**." The better an engineer's foundational skills, the greater the advantage gained from these tools.

## Update

A note added February 23, 2026 indicates "Agentic Engineering" has emerged as preferred terminology.
