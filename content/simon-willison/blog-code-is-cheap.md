# Writing Code is Cheap Now

**Date:** 23rd February 2026

## Overview

This article explores how AI-powered coding agents have fundamentally shifted the economics of software development, challenging long-established engineering practices.

## Key Arguments

**The Cost Paradigm Shift**

Traditionally, producing quality code required substantial time investment, shaping both organizational planning and daily development decisions. The article notes that "writing code is cheap now" because coding agents dramatically reduce the effort needed to generate code. This disrupts established intuitions about resource allocation and project trade-offs.

**Good Code Remains Expensive**

While initial code generation costs have plummeted, delivering truly "good code" remains resource-intensive. The author defines good code as having these attributes:

- Functional correctness with confirmed reliability
- Proper error handling and graceful degradation
- Simplicity and minimal design
- Comprehensive test coverage
- Current, accurate documentation
- Maintainable architecture respecting YAGNI principles
- Appropriate non-functional qualities (security, accessibility, scalability, etc.)

**Emerging Best Practices**

The article emphasizes that industry norms around agentic engineering remain unsettled. The recommendation: reconsider instincts to skip projects due to time constraints. Instead, task agents asynchronously with low-risk experiments, checking results later.

## Implications

This represents a philosophical shift—developers must reconceptualize which trade-offs warrant concern when machine-assisted code generation becomes the norm.
